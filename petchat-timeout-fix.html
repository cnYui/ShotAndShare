🐾 petChat云函数超时问题修复方案

❌ 问题描述

错误信息：

cloud.callFunction:fail Error: errCode: -504003 | errMsg: Invoking task timed out after 3 seconds FUNCTIONS_TIME_LIMIT_EXCEEDED

问题原因：

  • 云函数默认超时时间为3秒
  • DeepSeek API调用设置了10秒超时
  • 网络延迟导致API响应时间过长
  • 缺乏降级处理机制

✅ 解决方案

通过以下4个步骤彻底解决超时问题：

步骤1: 优化API调用超时时间

// 修改前：timeout: 10000 (10秒) // 修改后：timeout: 2000 (2秒) const response = await axios.post(DEEPSEEK_API_URL, { model: 'deepseek-chat', messages: messages, max_tokens: 100, // 从150减少到100 temperature: 0.8 }, { headers: { 'Authorization': `Bearer ${DEEPSEEK_API_KEY}`, 'Content-Type': 'application/json' }, timeout: 2000 // 减少到2秒 });
已完成

步骤2: 添加超时降级机制

// 添加超时错误处理 if (error.code === 'ECONNABORTED' || error.message.includes('timeout')) { return { success: true, data: getFallbackReply() // 返回预设回复 }; } // 预设回复函数 function getFallbackReply() { const fallbackReplies = [ '主人，我现在有点困，但还是很开心见到你！喵~', '网络有点慢呢，不过我一直在这里陪着你！', '虽然反应慢了点，但我对主人的爱不会变！汪~', '让我想想...主人今天过得怎么样呀？', '我的小脑袋瓜转得有点慢，但心里满满都是对主人的关心！' ]; return fallbackReplies[Math.floor(Math.random() * fallbackReplies.length)]; }
已完成

步骤3: 优化主函数错误处理

// 修改前：API失败直接返回错误 if (!response.success) { return { success: false, error: '大模型调用失败: ' + response.error }; } // 修改后：API失败使用降级回复 let petReply; if (!response.success) { console.warn('API调用失败，使用降级回复:', response.error); petReply = getFallbackReply(); } else { petReply = response.data; }
已完成

步骤4: 部署更新的云函数

# 部署命令 ./deploy-cloudfunctions.sh # 或在微信开发者工具中手动上传petChat云函数
已完成

🎯 修复效果

  • 响应时间：从可能的10秒+超时降低到最多2秒
  • 成功率：即使API调用失败，用户也能收到友好的回复
  • 用户体验：不再出现超时错误，聊天更加流畅
  • 降级策略：网络问题时自动使用预设回复

📱 测试方法

在小程序中测试宠物聊天功能：

  1. 打开小程序聊天页面
  2. 发送消息给宠物
  3. 观察响应时间（应在2秒内）
  4. 验证不再出现超时错误
  5. 即使网络慢也能收到回复

⚠️ 注意事项

  • 确保DEEPSEEK_API_KEY环境变量已正确配置
  • 如果仍有问题，可以进一步减少timeout到1秒
  • 预设回复会随机选择，保持对话的趣味性
  • 建议监控API调用成功率，优化网络配置
