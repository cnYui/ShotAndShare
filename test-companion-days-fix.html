🐾 陪伴天数修复验证指南

🔍 问题分析

根据数据库检查，发现pets集合中的记录缺少 created_at 字段，导致陪伴天数计算失败，始终显示为0。

// 原始数据结构（缺少created_at字段） { "_id": "9d458e0f688455c2001febad540e2d1c", "pet_name": "小绿", "user_id": "od5hLvLnBKWGTGM9F5GFvM_zb35I", "level": 3, "exp": 200, // ❌ 缺少 created_at 字段 "last_active": "Sun Jul 27 2025 10:11:40 GMT+0800" }

✅ 修复方案

已在 getPetStatus 函数中添加自动修复逻辑：

// 修复逻辑 if (!pet.created_at && pet._createTime) { // 使用_createTime作为创建时间 await db.collection('pets').doc(pet._id).update({ data: { created_at: new Date(pet._createTime) } }); } else if (!pet.created_at) { // 使用当前时间作为创建时间 const now = new Date(); await db.collection('pets').doc(pet._id).update({ data: { created_at: now } }); }

✨ 修复效果：首次调用getPetStatus时会自动为宠物添加created_at字段，后续计算陪伴天数将正常工作。
